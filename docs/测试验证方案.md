# 测试验证方案

说明：以下命令默认先进入仓库内的 `fullstack/` 目录执行。

## 1. 目标与范围

本方案用于支撑项目验收阶段的测试验证工作，重点验证以下内容：

- 核心业务逻辑的正确性、稳定性与边界处理能力
- 主要 API 接口在正常场景、异常场景、权限场景下的返回行为
- 关键状态流转链路在执行前后对系统数据的影响是否符合预期

本项目测试验证同时包含：

- 单元测试
- API 接口功能测试

两类测试均已纳入统一执行脚本，满足一键执行和可重复执行要求。

此外，项目已补充前端 E2E 联调测试，作为更高层的回归验证手段，详见 [前端E2E联调测试方案](./前端E2E联调测试方案.md)。

## 2. 测试目录结构

仓库中的测试与结果目录如下：

```text
fullstack/tests/unit_tests/
fullstack/tests/API_tests/
fullstack/tests/e2e_tests/
fullstack/scripts/run_tests.sh
fullstack/runtime/test_results/
```

说明如下：

- `fullstack/tests/unit_tests/`：存放单元测试清单、执行脚本和说明文件
- `fullstack/tests/API_tests/`：存放 API 接口测试清单、执行脚本和说明文件
- `fullstack/tests/e2e_tests/`：存放前端 E2E 测试执行脚本与环境启动脚本
- `fullstack/scripts/run_tests.sh`：统一测试入口脚本，可一次性组织全部测试执行

## 3. 单元测试覆盖说明

当前单元测试重点覆盖：

- 认证模块：登录失败锁定、账户锁定读取、刷新令牌拒绝非法状态用户
- 档案模块：修订字段比对、状态流转约束、上架状态补全、密级访问判断
- 借阅模块：超期天数计算、催还类型识别、超期状态同步、非法借阅申请校验
- 销毁模块：附件格式与大小校验、审批执行权限判定、销毁执行前置状态校验
- 扫描模块：扫描任务档案合法性校验、任务状态汇总、上传文件格式与大小校验
- 初始化模块：默认账号、角色、部门及账号文件生成逻辑

## 4. API 接口测试覆盖说明

当前 API 接口测试覆盖：

- 认证接口
- 用户、角色、权限、组织管理接口
- 档案主数据与实体位置接口
- 数字化扫描任务接口
- 借阅申请、审批、出库、归还、催还、通知接口
- 销毁流程接口
- 审计日志接口
- 报表接口
- 健康检查接口

当前重点补充的异常与边界测试包括：

- 销毁流程中的越权创建、越权审批、驳回缺少意见、非法附件执行
- 通知已读接口中的重复标记已读、越权标记他人通知、无未读时批量已读

当前前端 E2E 联调测试覆盖：

- 登录鉴权回跳
- 角色菜单与受限页面访问控制
- 审计员报表访问
- 借阅申请、审批、出库、归还、通知回跳闭环
- 销毁申请、审批、执行、通知回跳闭环

## 5. 执行方式

### 5.1 统一执行全部测试

```bash
./scripts/run_tests.sh
```

### 5.2 仅执行单元测试

```bash
./tests/unit_tests/run_unit_tests.sh
```

或：

```bash
./scripts/run_tests.sh unit
```

### 5.3 仅执行 API 接口测试

```bash
./tests/API_tests/run_api_tests.sh
```

或：

```bash
./scripts/run_tests.sh api
```

### 5.4 仅执行前端 E2E 联调测试

```bash
./scripts/run_tests.sh e2e
```

### 5.5 执行完整测试链路

```bash
./scripts/run_tests.sh full
```

## 6. 输出结果

测试执行过程中会输出：

- 当前测试集名称
- 测试标签
- 执行日志文件路径
- 测试执行成功或失败状态
- 失败场景下的末尾错误日志摘要

测试执行完成后会生成：

- `fullstack/runtime/test_results/summary.log`：整体测试汇总
- `fullstack/runtime/test_results/unit_tests/*.log`：单元测试详细日志
- `fullstack/runtime/test_results/API_tests/*.log`：API 测试详细日志
- `fullstack/runtime/test_results/e2e_tests/*.log`：前端 E2E 执行日志
- `fullstack/runtime/test_results/e2e_tests/report.json`：E2E 结构化测试报告
- `fullstack/runtime/test_results/e2e_tests/html-report/`：E2E 可视化报告

## 7. 验收判定依据

验收人员可通过执行统一脚本并检查输出结果，确认：

- 单元测试与 API 测试均已完整执行
- 关键模块与主要业务路径具备测试覆盖
- 失败场景下能清晰定位失败原因与对应日志
- 测试结果汇总信息完整、可读、可复核
