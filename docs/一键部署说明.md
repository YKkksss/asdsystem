# 一键部署说明

说明：以下命令默认先进入仓库内的 `fullstack/` 目录执行。

## 目标

当前项目已经支持 `fullstack/` 开发目录内的一键部署，目标是让用户在不手工修改配置文件的情况下，直接完成：

1. 基础设施启动
2. 前后端服务启动
3. Celery Worker / Beat 启动
4. 数据库迁移
5. 基础角色、部门、示例账号初始化

## 推荐命令

```bash
./scripts/deploy.sh
```

前置条件：

1. 服务器已安装 Docker
2. 服务器已安装 Docker Compose Plugin
3. 默认访问端口 `8080` 已放行

如果你不想使用包装脚本，也可以直接执行：

```bash
docker compose -f docker/docker-compose.deploy.yaml up -d --build
```

## 自动完成内容

`fullstack/docker/docker-compose.deploy.yaml` 已包含以下服务：

1. `mysql`
2. `redis`
3. `backend-init`
4. `backend`
5. `worker`
6. `beat`
7. `frontend`

其中 `backend-init` 会自动完成：

1. 等待 MySQL 与 Redis 就绪
2. 执行 `manage.py migrate`
3. 执行 `manage.py bootstrap_system`
4. 生成账号清单文件到 `runtime/deployment_runtime/accounts.md`

## 默认账号

默认初始化账号说明如下：

- 管理员账号固定为 `admin`，密码优先取 `ASD_ADMIN_PASSWORD`。
- 如果未设置 `ASD_ADMIN_PASSWORD`，部署流程会自动生成随机密码，并写入 `runtime/deployment_runtime/accounts.md`。
- 固定示例账号：
- 档案员：`archivist / Archivist12345`
- 借阅人：`borrower / Borrower12345`
- 审计员：`auditor / Auditor12345`

部署完成后，请以 `runtime/deployment_runtime/accounts.md` 中实际生成内容为准。

## 默认访问地址

1. 前端首页：`http://127.0.0.1:8080`
2. 后端健康检查：`http://127.0.0.1:8080/api/v1/system/health/`
3. 其他设备访问：`http://服务器IP或域名:8080`

## 常用管理命令

```bash
./scripts/deploy.sh status
./scripts/deploy.sh logs
./scripts/deploy.sh logs backend
./scripts/deploy.sh down
./scripts/deploy.sh restart
```

## 可选环境变量

如果需要覆盖默认值，可在执行命令前设置：

```bash
ASD_HTTP_PORT=8080
ASD_ADMIN_USERNAME=admin
ASD_ADMIN_PASSWORD=MyAdmin123
DJANGO_ALLOWED_HOSTS=demo.example.com,127.0.0.1
DB_NAME=lanchang_archive
DB_USER=archive
DB_PASSWORD=archive123456
MYSQL_ROOT_PASSWORD=root123456
```

示例：

```bash
ASD_HTTP_PORT=8080 ASD_ADMIN_PASSWORD=MyAdmin123 DJANGO_ALLOWED_HOSTS=demo.example.com,127.0.0.1 ./scripts/deploy.sh
```

## 适用边界

本方案定位是“可直接试用、可直接验收、可直接演示”的一键部署方案，适合：

1. 本地试用
2. 演示环境
3. 验收环境

如果是正式生产环境，仍建议结合 `fullstack/deploy/` 目录中的 `systemd`、`supervisor`、`nginx`、`logrotate` 模板做更严格的部署治理。

补充说明：

1. 当前一键部署默认关闭 `DJANGO_DEBUG`，并将 `DJANGO_ALLOWED_HOSTS` 收紧为 `127.0.0.1,localhost,backend`
2. 前端 Nginx 已固定代理上游 Host，因此本机和常见内网通过前端入口访问时无需额外修改
3. 如果进入正式生产环境，建议显式设置 `DJANGO_ALLOWED_HOSTS` 为实际域名或 IP 白名单
4. 默认 `DJANGO_CORS_ALLOW_ALL=false`，生产场景如需跨域，请显式补充 `DJANGO_CORS_ALLOWED_ORIGINS`

## 当前验证结果

已在当前仓库所在环境完成以下验证：

1. `./scripts/deploy.sh` 可完成镜像构建、容器启动、迁移和基础数据初始化
2. `runtime/deployment_runtime/accounts.md` 会自动生成默认账号清单
3. `http://127.0.0.1:8080` 可返回前端首页
4. `http://127.0.0.1:8080/api/v1/system/health/` 可返回后端健康检查结果
5. 管理员账号 `admin` 可登录，密码以部署生成的账号清单为准
