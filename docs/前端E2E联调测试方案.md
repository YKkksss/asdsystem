# 前端E2E联调测试方案

说明：以下命令默认先进入仓库内的 `fullstack/` 目录执行。

## 1. 目标

本方案用于补充单元测试与 API 接口测试之外的真实联调验证，重点覆盖前端路由、登录鉴权、角色权限跳转、工作台真实业务加载能力与关键业务闭环。

## 2. 技术方案

- 测试框架：Playwright
- 前端运行方式：Vite 开发服务
- 后端运行方式：Django 本地服务
- 测试数据准备：通过 `bootstrap_system` 初始化默认角色账号，并在用例执行时通过后端接口创建联调用档案与实体位置

## 3. 当前覆盖场景

### 3.1 管理员登录回跳链路

- 未登录直接访问系统管理页
- 自动跳转到登录页
- 登录成功后自动回跳系统管理页
- 页面可正常加载用户管理标签
- 可正常退出登录并回到登录页

### 3.2 角色菜单与页面权限链路

- 档案员登录后仅展示工作台、档案业务、借阅业务、统计监督等允许访问的父级菜单
- 档案员直接访问借阅审批、系统管理等越权页面时自动回退到工作台
- 借阅人登录后仅保留个人借阅闭环相关菜单
- 借阅人直接访问档案中心、系统管理等越权页面时自动回退到工作台
- 审计员可进入报表中心、审计日志，但不能进入档案中心与销毁中心

### 3.3 工作台待办直达链路

- 工作台需按角色显示真实业务指标与待办项
- 管理员可从工作台待办明细直达借阅审批弹窗
- 档案员可从工作台待办明细直达出库登记弹窗
- 借阅人可从工作台待办明细直达归还提交弹窗

### 3.4 借阅业务闭环链路

- 通过后端接口初始化可借阅档案和实体位置
- 借阅人发起借阅申请并查看详情状态
- 管理员完成审批通过与出库登记
- 借阅人提交归还附件
- 管理员完成归还确认与重新入库
- 借阅人从通知中心回跳到业务详情页并确认流程已闭环

### 3.5 销毁业务闭环链路

- 通过后端接口初始化已上架档案和实体位置
- 档案员通过前端页面发起销毁申请并查看详情状态
- 管理员完成销毁审批通过
- 档案员上传销毁附件并完成执行登记
- 档案员从通知中心回跳到销毁详情页并确认流程已闭环

### 3.6 文件访问权限与脱敏链路

- 通过后端接口初始化已数字化档案、扫描任务和文件资源
- 有权限用户可看到在线预览、下载文件、文本提取等入口
- 密级不足用户仍可检索到档案，但详情页只显示后端裁剪后的脱敏字段
- 无权限场景下不展示“在线预览”“下载文件”等敏感入口
- 页面需明确提示当前账号因密级不足而被限制访问，避免白屏或无反馈

### 3.7 通知深链与业务回跳链路

- 通知中心根据 `notificationId` 聚焦目标通知行
- 借阅通知可从通知中心回跳到借阅业务详情
- 销毁审批通知可从通知中心回跳到销毁申请详情
- 报表导出通知可从通知中心回跳到报表中心
- 通用系统通知无业务对象时，仍需回到通知中心本页并隐藏“查看业务”按钮

### 3.8 页面深链与精准定位链路

- 系统管理页根据 `tab` 与 `userId` 直达用户管理标签并自动打开编辑弹窗
- 档案中心根据 `archiveId` 与 `fileId` 直达目标档案详情并高亮目标文件
- 扫描任务详情根据 `itemId` 聚焦指定扫描明细卡片

## 4. 执行方式

```bash
./tests/e2e_tests/run_e2e_tests.sh
```

或：

```bash
./scripts/run_tests.sh e2e
```

如需执行完整测试链路：

```bash
./scripts/run_tests.sh full
```

## 5. 输出结果

执行后会生成：

- `fullstack/runtime/test_results/e2e_tests/01_playwright_install.log`
- `fullstack/runtime/test_results/e2e_tests/02_playwright_run.log`
- `fullstack/runtime/test_results/e2e_tests/report.json`
- `fullstack/runtime/test_results/e2e_tests/html-report/`

## 6. 说明

- E2E 联调测试会自动启动独立的前后端测试服务，不依赖当前手工启动的开发进程。
- 后端测试环境使用独立 SQLite 数据文件，避免污染常规开发数据库。
